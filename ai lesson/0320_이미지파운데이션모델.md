# AI 파운데이션 모델의 개념
## 파운데이션 모델(Foundation model)이란?
- AI 모델
  - 입출력을 연결해주는 함수 + 데이터로 학습된 함수 + <mark>학습 때 보지 못했던 데이터에 대해서도 작동</mark>해야하는 의무

- 현실적인 기계학습 모델
  - AI모델에 데이터를 패턴화하여 압축

- 파운데이션 모델이란?
  - 대규모 데이터를 폭넓게 학습한 후, 다양한 문제에 빠르게 적응할 수 있는 범용 대형 AI모델

- 파운데이션 모델
  - 기존 딥러닝 개발 패러다임: 아기와 같이 언어, 시각, 청각, 촉각 등 기본적인 것들부터 배워 나가야 함
  - 파운데이션 모델 패러다임: 거대 모델(커다란 뇌) + 대규모 데이터 학습 (많은 지식과 경험) 기반

- 파운데이션 모델의 특징
  1. 대규모: 트랜스포머 모델 + 대규모 언어 데이터 학습
  2. 적응성: 높은 파인튜닝 성능(높은 테스크 적응 성능)
  3. 범용성: 다양한 작업, 한정되지 않는 출력 지원

# 대표적 AI 파운데이션 모델: CLIP
- CLIP(Contrastive Language-Image Pre-training)
  - AI가 언어와 시각을 통합해서 이해하는 방식을 보여준 패러다임 전환 제시
  - 파운데이션 모델로써의 특징
    - 입력: 학습하지 않은 새로운 도메인의 입력 데이터에 대해서도 좋은 성능을 발휘(제로샷 전이)
    - 출력: 자연어를 이용해 한번도 본 적 없는 카테고리도 텍스트 설명만으로 출력 정의 가능(언어 인터페이스)

- 대조 학습 기반(Contrastive Pre-training)의 언어-이미지 사전 학습
  - 인터넷 데이터를 통한 지도 학습을 통해 자연어 기반 시각 개념 학습
  - 다양한 이미지-자연어 쌍으로 학습

- CLIP 구조: 텍스트 인코더(Tranformer 기반 Text Encoder)
  - 트랜스포머 구조: 인코더 + 디코더
  - CLIP에서는 Encoder only구조 사용

- CLIP 학습
  - 대조 학습
  - 학습 기준
    - 목표 이미지(앵커)를 대응하는 텍스트(양성)와 가깝게
    - 일치하지 않는 여러 텍스트(음성)와는 멀게

- SigLIP
  - SigLIP은 softmax 대신 sigmoid기반 손실함수
    - 기존 CLIP에서 학습한 대조학습의 한계: 어느 정도 이미 멀게 배치한 음성 데이터들에 대해서도 계속 거리를 벌리기 위해 학습이 진행됨
  - SigLIP: CLIP과 달리 일치하지 않는 음성 데이터에 제한된 영향만 받도록 손실함수 디자인을 고침

- 멀티모달 정합 응용
  - 멀티모달 정합(Multi-modal Alignment)
    - 서로 다른 두 가지 이상의 모달리티간의 공통된 임베딩 벡터 공간을 구성하는 것
    - 서로 다른 모달리티 임베딩 간 유사도(연관성) 비교 가능

# Vision-Language Models(VLM)
## 멀티모달 언어 모델
- 이미지, 소리, 비디오 등 다양한 모달리티를 함께 이해하고 처리할 수 있는 언어 모델

## LLaVA(Large Language and Vision Assistant)
- Vision과 Language모델을 결합한 모델(VLM)로, 텍스트와 이미지를 동시에 이해
- LLaVA모델 특징
  - 효율적인 메모리 사용: 적은 자원으로 큰 모델을 효과적으로 학습
  - 다중 모달 학습: 텍스트와 시각 데이터를 결합하여 응답을 생성
  - Fine-tuning: 특정 작업에 맞춰 모델을 미세조정하여 사용

# 개인화, 합성 데이터 활용 사례
## 파운데이션 모델 + Fine-tuning
- 파운데이션 모델과 미세조정이 필요한 이유
  - 방대한 데이터로 학습된 초대형 딥러닝 모델. 다양한 작업이나 범용적인 문제에 바로 적용 가능
  - 최근에는 텍스트 뿐만 아니라 이미지, 오디오, 비디오 등의 다양한 입력 데이터를 처리할 수 있는 멀티모달로 확장
  - 하지만 최신 정보나 특정한 작업/도메인에 최적화 되어 있지 않아, 즉시 활용이 어려운 경우가 반드시 있음

## Fine-tuning이란?
- 추가 학습을 통해 이미 학습된 모델을 조금만 튜닝하는 것
  - 미세조정을 통해 특정 작업에 특화된 모델 개발 가능
  - 프롬프트 길이 줄어들면서 토큰 개수 절약
  - 응답하는 데에 걸리는 시간(latency) 단축

## 하이퍼파라미터 - Learning Rate
  -  미세조정에서는 좋은 시작점에서부터 시작하기 때문에 작은 learning rate부터 보수적으로 시작해야 함

## Parameter-Efficient Fine-Tuning(PEFT)
- 하드웨어 발전, 대규모 데이터 축적, AI 모델 발전에 따라 학습 비용과 모델 용량 기하급수적으로 증가 -> fine tuning도 부담
- 프롬프트 디자인(prompt design)
  - 언어모델에서 주로 활용. 모델이 원하는 레벨의 결과를 출력할 수 있도록 입력 텍스트를 변형하는 방법
  - 장점: 추가 학습 없이 사전학습된 모델의 예측 성능을 끌어올릴 수 있음
  - 단점: 프롬프트를 사람이 직접 설계해야 한다는 부담이 있으며 성능 향상이 제한적

- 프롬프트 튜닝(prompt tuning)
  - 학습 가능한 프롬프트로서, 가상 토큰(virtual token)을 입력에 추가
  - 역전파를 통해 오직 가상 토큰에 대한 임베딩만 학습하고 나머지 모델은 고정
  - 장점
    - 사람 디자인 없이 스스로 프롬프트를 학습
    - 사전학습된 모델 고정 가능(지식 손실 X; 일반 파인튜닝은 지식 손실 발생)
    - 적은 비용을 ㅗ새로운 데이터셋의 모델 학습 가능
  - 학습된 프롬프트는 해석 X

- Adaptor 모듈 추가 학습
  - 학습 가능한 작은 모듈(Adapter)을 레이어마다 삽입하여 일부 파라미터만 학습하는 기법

# 데이터 효율화를 위한 합성 데이터 사용
## 합성데이터 활용법1
- Knowledge Distillation(Teacher-Student 학습)
  - 높은 성능의 무거운 모델(선생님)을 모방하도록 가벼운 모델(학생)을 학습하는 방법

## 합성데이터 활용법2
- 파운데이션 모델들을 툴로 활용하는 방법: InstructPix2Pix
  - 파운데이션 모델을 이용해 이미지-명령 쌍 데이터셋을 생성하여 최종 이미지 편집 생성 모델을 학습
  - InstructPix2Pix: 입/출력 이미지 상세 설명 없이, 명령만으로 편집 수행

## 합성데이터 활용법3
- 간단한 시뮬레이션 기반 합성 데이터: 실제 데이터를 모방하거나 새로 생성한 인공 데이터
  - ex: 가상의 이미지, 텍스트, 소리 등을 알고리즘을 통해 생성
