# Intro & Design Pattern
## Web Application
### 클라리언트와 서버
- client: 서비스를 요청하는 주체
- server: 클라이언트의 요청에 응답하는 주체
### Frontent & Backend
- Frontent: UI를 구성하고, 사용자가 애플리케이션과 상호작용할 수 있도록 함
- Backend: 서버 측에서 동작하며, 클라이언트의 요청에 대한 처리와 DB와의 상호작용 등을 담당

## 가상 환경
- 가상환경: 하나의 컴퓨터 안에서 또 다른 독립된 파이썬 환경
### 가상 환경 생성 및 활성화
- 생성: python -m venv venv
- 활성화: source venv/Scripts/activate
- 종료: deactivate
### 의존성 패키지
- 의존성: 하나의 소프트웨어가 동작하기 위해 필요로 하는 다른 소프트웨어나 라이브러리
- 의존성 패키지: 프로젝트가 의존하는 개별 라이브러리들을 가리키는 말
- 패키지 목록 확인: pip list
### 의존성 관리: 환경의 일치
- 의존성 기록(내보내기): pip freeze > requirements.txt
- 의존성 설치(가져오기): pip install -r requirements.txt
### 가상 환경 주의사항
- 프로젝트마다 별도의 가상 환경을 사용
- 일반적으로 가상 환경 폴더는 관련 프로젝트와 동일한 경로에 위치시킴
- venv는 .gitignore파일에 작성되어 원격 저장소에 공유하지 않음

## Django Design Pattern
### Design Pattern
- 소프트웨어 설계에서 반복적으로 발생하는 문제에 대한, 검증되고 재사용 가능한 일반적인 해결책
- MVC 디자인 패턴
  - Model: 데이터 및 비즈니스 로직을 처리
  - View: 사용자에게 보이는 화면을 담당
  - Controller: 사용자의 입력을 받아 Model과 View를 제어
- MTV 디자인 패턴
  - Django에서 애플리케이션을 구조화하는 디자인 패턴
  - View -> Template, Controller -> View : 이름만 다르게 부름
### 프로젝트와 앱
- 프로젝트: 애플리케이션의 집합
- 애플리케이션: 독립적으로 작동하는 기능 단위 모듈
- 앱 생성: python manage.py startapp articles
- 앱 등록: settings.py에 installed_apps에 'articles' 추가
