# 워드 임베딩과 순환신경망 기반 모델(RNN & LSTM)
1. 워드 임베딩
  1. 원-핫 인코딩
    - 단어를 한 원소만 1이고 나머지는 모두 0인 벡터(원-핫 표현)로 바꾸는 과정
    - 문제점
      - 고차원의 벡터를 다루기 위해선 많은 메모리 필요, 차원 커질수록 데이터 희소해져 활용 어려움
      - 비슷한 단어라도 유사한 벡터로 표현되지 않는다

  2. 워드 임베딩
    -  단어를 단어들 사이의 의미적 관계를 포착할 수 있는 밀집(dense)되고 연속적/분산적(distributed) 벡터 표현으로 나타내는 방법

2. 순차적 데이터
- 순데이터가 입력되는 순서와 이 순서를 통해 입력되는 데이터들 사이의 관계가 중요한 데이터
- 특징
  - 순서 중요: 데이터 순서가 바뀌면 의미 달라짐
  - 장기 의존성: 멀리 떨어진 과거의 정보가 현재/미래에 영향
  - 가변 길이: 길이가 일정하지 않고, 단어 수도 제각각
- Sequential Models(RNN, LSTM, Transformer 등)으로 처리

3. RNN(Recurrent Neural Network)
- 가변 길이의 입력 받을 수 있고, 이전 입력 기억할 수 있기 때문에 순차적 데이터 처리에 적합한 아키텍처
- 전통적인 신경망(MLP, CNN 등)과 달리 이전 시점의 정보를 담는 hidden state를 가짐
- 입력 시퀀스 벡터 x를 처리할 때, 각 시점마다 recurrence 수식을 적용하여 hidden state를 업데이트
- 한계: 기울기 소실(vanishing gradient) 문제
    - 기울기 소실: 딥러닝에서 역전파 시 앞쪽 층의 기울기가 0에 가까워져 장기 의존성 학습이 어려워지는 현상
  
4. LSTM(Long Short-Term Memory)
- 기울기 소실 문제를 해결하기위해 제안된 RNN의 한 종류
- 특징
  - 시점 t에서 RNN은 길이가 n인 벡터 hidden state(ht)와 cell state(Ct)를 가진다.
    - ht는 short-term information을 저장
    - Ct는 long-term information을 저장
  - LSTM은 cell state에서 정보를 읽고(read), 지우고(erase), 기록(write)할지 3가지 게이트를 통해 결정한다.
    - Forget gate: 이전 Ct에서 무엇을 버리고 무엇을 유지할지 결정
    - Input gate: 새 정보 중 얼마나 Ct에 쓸지 결정
    - Output gate: Ct 중 얼마나 hidden state로 내보낼지 결정

# 자연어 생성 모델(Seq2Seq, Attention)
1. 언어 모델이란?
- 인간의 두뇌가 자연어를 생성하는 능력을 모방한 모델
  - 단어 시퀀스 전체에 확률을 부여하여 문장의 자연스러움을 측정
  - 한 문장의 확률은 각 단어의 조건부 확률들의 곱으로 표현

- 대표적인 언어모델: N-gram 언어모델
  - N-gram: 연속된 n개의 단어 묶음
  - 다양한 N-gram이 얼마나 자주 등장하는지 통계 수집하고, 이를 활용해 다음 단어를 예측

- 언어모델 사용 예시: SMT(Statistical Machine Translation)
  - Machine Translation을 통계학적으로 접근
  - 한계
    - 구조적 복잡성
    - 많은 수작업
    - 언어 별 자원 구축 필요

2. Seq2Seq
- Neural Machine Translation(NMT)이란?
  - 인공 신경망을 이용해 기계 번역을 수행하는 방법
  - 이 때 사용되는 신경망 구조를 Seq2Seq라 하며, 두개의 RNNs로 이루어짐

- Translation 어려운 이유
  - 번역 문제는 입력과 출력의 길이가 다를 수 있다.
- Seq2Seq의 아이디어: 2개의 LSTM을 이용하자
  - 한 LSTM은 입력 시퀀스를 한 타임스텝씩 읽어 고정된 차원의 큰 벡터 표현을 얻기(Encoder)
  - 다른 LSTM은 앞에서 얻은 벡터로부터 출력 시퀀스 생성하기(Decoder)

- Seq2Seq Architecture: Encoder + Decoder
  - Encoder는 입력 문장에 담긴 정보를 인코딩
  - Decoder는 인코딩된 정보를 조건으로 타겟 문장(출력)을 생성

- 학습 수행
  - 디코더에서 발생한 오차는 역전파 과정을 통해 입력을 처리한 인코더까지 전달되어 전체 네트워크가 End-to-End로 동시에 최적화된다.
  - Teacher Forcing이란?
    - 모델이 스스로 예측한 단어 대신 정답 단어를 디코더 입력으로 강제로 넣어줌으로써 훨씬 안정적이고 빠르게 학습을 수행하는 방법.

- 토큰 출력 방법(Greedy Inference)
  - 각 단계에서 가장 확률이 높은 단어를 선택
  - 한계
    - 되돌리기가 불가능

  - Beam Search
    1. 매 단계마다 k개의 가장 유망한 후보 유지
    2. 후보가 EOS에 도달하면, 완성된 문장으로 리스트 추가
    3. EOS문장이 충분히 모이면 탐색 종료
    4. 각 후보들의 점수를 로그 확률의 합으로 구해 최종 선택

3. Attention
- Seq2Seq의 한계: bottleneck problem
  - 인코더는 입력 문장 전체를 하나의 벡터로 요약하는데, 마지막 hidden state에 문장의 모든 의미 정보가 담긴다.
  - 고정 길이 벡터 하나에 모든 문장의 의미를 압축하다 보니 정보 손실이 생길 수 있는데, 이를 bottleneck problem이라 한다.

- Attention의 인사이트
  - 디코더가 단어를 생성할 때, 인코더 전체 hidden state 중 필요한 부분을 직접 참조할 수 있도록 한다.
  - 매 타입스텝마다 어떤 단어/구절에 집중할지 가중치로 계산해 bottleneck 문제를 완화
  - 효과
    - NMT 성능 향상: 디코더가 소스 문장 전체가 아닌, 필요한 부분에만 집중
    - Bottleneck Problem 해결: 디코더가 인코더의 모든 hidden states에 직접 접근
    - Vanishing Gradient Problem 완화: 멀리 떨어진 단어도 직접 연결
    - 해석 가능성(Interpretablility): Attention 분포를 보면, decoder가 어떤 단어 생성할 때 입력 문장의 어느 부분에 집중했는지 확인 가능
    - 정렬(Alignment): decoder가 필요한 입력 단어에 자동으로 집중하기 때문에, 단어와 단어 간 매핑 관계 자연스럽게 학습

# Transformer
1. Self-Attention
- Attention은 각 단어의 표현을 query로 두고, value 집합으로부터 필요한 정보를 직접 불러와 결합
- 이러한 메커니즘을 encoder-decoder 간이 아닌, 한 문장 내부에서 적용한다면? -> Self-Attention
- 장점
  - 순차적으로 처리해야 하는 연산 수가 시퀀스 길이에 따라 증가하지 않음
  - 모든 단어가 각 층에서 직접 상호작용(병렬처리 가능)
- 한계
  - 순서 정보 부재: 단어 간 유사도만 계산
  - 비선형성 부족: 가중 평균 연산이라는 선형 결합에 불과하기 때문에 복잡한 패턴이나 깊은 표현력 담기 어려움
  - 미래 참조 문제: 모든 단어를 동시에 보기 때문에, 아직 생성되지 않아야 할 미래 단어를 참조

- 한계 해결
  - Positional Encoding
    - 순서 정보 부재 문제를 해결하기 위해 각 단어 위치 i를 나타내는 위치 벡터를 정의해, 단어 임베딩 값에 더해 최종 입력으로 사용
  - Feed-Forward Network 추가
    - 각 단어 출력 벡터에 Feed-Forward Network(Fully Connected + ReLU 등)을 추가해 Self-Attention이 만든 표현을 깊고 비선형적인 표현으로 확장
  - Masked Self-Attention
    - Attention Score를 계산할 때, 미래 단어에 해당하는 항목을 -무한대로 설정해, 계산을 수행할 때 반영되지 않도록 한다.

2. Transformer
- Self-Attention을 핵심 메커니즘으로 하는 신경망 구조
- encoder-decoder 구조로 설계된 신경망 모델
- Multi-Headed Attention
  - 문장에서 같은 단어라도 여러 이유로 다른 단어에 주목할 수 있다. -> 여러 Attention Head를 두어 다양한 관점에서 동시에 정보를 파악한다.
- Scaled Dot Product
  - Query와 Key의 차원이 커질수록 두 벡터의 내적 값도 커진다.
  - 이 값이 너무 크면 softmax 함수 출력이 뾰족해져 미세한 변화에도 큰 차이 발생하고 gradient vanishing문제 발생 -> 내적 값을 나눠주어 스케일 조정
- Residual Connection
  - 깊은 신경망은 층이 깊어질수록 학습 어려워진다 -> 기존 입력과의 차이만 학습하도록 하는 residual connection을 사용
- Layer Normalization
  - 층이 깊어질수록, hidden vector 값들이 커졌다 작아졌다 하면서 학습 불안정 -> 각 레이어 단위에서 hidden vector 값을 정규화해, 안정적이고 빠른 학습 도움

- Transformer와 사전학습(pretraining)
  - Transformer의 등장은 대부분의 최신 모델들이 성능 향상을 위해 사전학습을 결합하도록 했다.
  - 뛰어난 병렬 처리 능력 덕분에 대규모 사전학습에 적합하여 NLP의 표준 아키텍처로 자리 잡았다.
