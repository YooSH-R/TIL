# AI 모델 활용 "가속기 구동을 위한 100% 정수연산 양자화"
## 1. 100% 정수연산 양자화의 필요성
### 1-1. AI 가속기의 구동 특성
- AI 가속기는 정수 연산만 지원하거나, 정수 연산에서 훨씬 효울적으로 동작함

### 1-2. 일반 양자화와 100% 정수연산 양자화의 차이
- 일반적 양자화: r이라는 실수 값을 q라는 정수로 표현: r= S(q - Z)
- 문제는 스케일링 파라미터가 실수라는 것
<br><br>

## 2. 100% 정수연산 양자화
### 2-1. 행렬 곱 연산 양자화
- 스케일링 파라미터를 정수화하여 q3(i, k)를 정수 연산으로 유도: 스케일링 파라미터를 변수 한 개로 묶음
- 비트 쉬프팅: 스케일링 파라미터를 뭉쳐서 M으로 나타냈으므로, M을 별도로 미리 양자화시키면 실수 연산 전멸
  - 임시로 스케일 업 -> 양자화 및 연산 -> 스케일 다운

### 2-2. 배치 정규화 계층 폴딩
- 배치 정규화(BN): 학습 중 정규화를 통해 입력 분포의 변화를 줄여 안정적인 학습을 돕는 기법
- BN 계층에서 별도로 양자화를 진행하면 비효율적임 -> 폴딩 기법 사용
- 배치 정규화 계층 폴딩: BN 연산을 바로 앞의 Conv 계층 연산에 통합해서 함께 양자화

### 2-3. 비전 트랜스포머 연산 양자화
- 행렬 곱과 배치 정규화 폴딩 외에 추가적 연산 정수화가 필요
- SoftMax
- GELU
- LayerNorm 클라이밍 가자고요

# AI 모델 활용 "테스트 타임 도메인 적응"
## 1. 분포 이동과 테스트 타임 도메인 적응(TTA)
- 분포 이동(Distribution Shift)
  - AI 모델이 현실에서 동작할 때, 학습 데이터에서 경험하지 못했던 낯선 데이터에 맞닥뜨리는 현상
  - AI 성능 하락의 주요 원인

- 테스트타임 도메인 적응(Test-Time Adaptation: TTA)
  - AI가 실제 쓰이는 순간, 테스트 데이터의 분포를 파악하여 모델을 유연하게 적응시킴
  - 즉, 테스트 데이터를 입력받을 때 모델의 일부를 새 데이터의 통계에 맞게 재조정하여 성능을 유지하는 기법
<br><br>

## 2. TTA 기법
### 2-1. CNN 기반 TTA 기법
- TENT: 배치 정규화 계층을 테스트 데이터에 최적화
  - 테스트 데이터의 엔트로피를 최소화
  - 평균 분산 업데이트
  - 어파인 파라미터(스케일, 이동 값) 업데이트
  - 학습 재진행 없이, 테스트 데이터 분포에 맞게 모델을 즉시 최적화

- SAR: 최적화를 더욱 안정적으로
  - 관찰: 일부 테스트 데이터가 노이즈가 많거나 이상치라서 큰 gradient를 만들어 모델을 불안정하게 만듦
  - 테스트 샘플 선택: 엔트로피(불확실성)가 낮은 테스트 샘플만 선택해 모델 업데이트에 반영
  - 안정적 엔트로피 최소화: 엔트로피 손실이 너무 요동치지 않도록 평탄화(loss smoothing)
  - 결과: TENT보다 학습이 안정적이고, 테스트 환경에서도 성능 유지

### 2-2. 비전언어모델 기반 TTA 기법
- TPT(Test-Time Prompt Tuning): 일관성 있는 예측을 돕자
  - 테스트 이미지마다 여러 증강(회전, 밝기 변화 등)을 생성
  - 엔트로피가 가장 낮은 증강 이미지를 6개 선택
  - 6개의 평균 엔트로피를 로스로 세팅하고 이를 최소홯하도록 텍스트 프롬프트 업데이트

- PromptAlign: 학습 데이터와 테스트 데이터를 직접 정렬 시키자
  - PromptAlign은 학습 데이터 통계까지 활용해 분포 정렬(distribution alignment)을 수행
  - 학습 데이터와 테스트 데이터의 통계척 차이(평균, 분산)를 최소화하도록 설계
  - 두 데이터의 인코딩 결과를 계층별(feature layer)로 정렬
    - 학습 시 얻은 feature 분포와 테스트 feature 분포를 정렬시키는 방식
  - 추가적으로 엔트로피 로스를 함께 사용하여 안정적인 예측 유지
  - 최종적으로 텍스트 프롬프트 + 이미지 프롬프트를 모두 업데이트
<br><br>

# AI 모델 활용 "적응적 센싱"
## 1. 초거대 AI의 근본 원리와 한계
### 1-1. 스케일링의 법칙과 초거대 AI
- 초거대 AI
  - 파라미터 규모가 매우 큰 인공지능 모델
  - 데이터가 많고, 모델이 클수록 성능이 좋아짐: 스케일링 법칙
  - 분포 이동의 리스크를 줄이기 위해 규모로 보험을 들었음

### 1-2. 스케일링 전략의 한계
- 이 세상에서 일어날 수 있는 모든 경우의 수를 학습 데이터로 미리 준비하는 것이 가능할까?
  - AI의 진화: 거대 언어모델, 비전언어모델에서 피지컬 AI로
  - 피지컬 AI에서 분포 이동 문제의 심화(폭발하는 경우의 수)
  - 보완책의 필요성: 적응적 센싱이 필요함
<br><br>

## 2. 적응적 센싱을 통한 분포 이동 억제
### 2-1. 발상의 전환: 인간의 인지체계
- AI 거대화 전략과 인간 인지체계의 괴리
  - 초거대 AI의 발상: 모든 것은 공부/노력의 문제야
  - 문제 제기: 이 문제 해결 방식이 상식적인가? 정작 사람은 이렇게 세상을 인지하고 있는가?

### 2-2. 적응적 센싱: AI를 위한 안경
- 적응적 센싱의 개념
  - 적응적 센싱: 환경에 맞춰서 센서(카메라)를 조절해서 좋은 데이터만 받음
  - 분포 이동 억제: 모델 일반화 VS 적응적 센싱

### 2-3. 적응적 센싱을 위한 데이터셋
- 같은 장면을 카메라 센서 파라미터와 조도를 변화시키며 캡쳐

### 2-4. 적응적 센싱 기법: Lens
- 센서 파라미터 후보 선택 -> 이미지 캡쳐 -> 비전 테스트 -> 최적의 파라미터 선택

### 2-5. 성능 효과
- Lens vs. Auto-Exposure(AE)
  - Lens: 센서 제어로 모델이 보기 좋은 이미지를 만듦
  - AE: 사람이 보기 좋은 이미지를 만들기 위한 제어 기법
  - Lens의 효과: 정확도 향상 -> 모델만 키우는게 다가 아니고, 센서도 중요하다

### 2-6. 특징과 함의
- AI 모델이 더 잘 맞추는 사진
  - 적응적 센싱의 필요성: 최적의 센서 파라미터는 모델에 따라, 환경에 따라, 피사체에 따라 달라짐

# AI 모델 활용 "응용 분야 전문 지식을 활용한 모델 설계"
## 1. 도메인 전문지식의 필요성
- 도메인 특화 AI모델을 제작하고자 한다면, 그 도메인의 전문지식을 이해해야 함

## 2. 의료 응용: 수면 의학을 위한 AI
### 2-1. 수면의학의 기본적 이해
- 수면의학의 중요성
- 수면질환 진단의 기본요소: 수면다원검사

### 2-2. 수면다윈검사의 한계
- 전문 수면기사의 수동 채점 방식
- 수면다원검사의 근본적 문제
  - 첫날 밤 효과: 샘플의 부정확성
  - 하룻 밤 문제: 샘플 부족 문제

## 3. 개발 사례 1: 수면다원검사 자동 채점 AI
### 3-1. 수면단계 진단 AI의 필요성
- 수면단계
  - Wake, N1, N2, N3, REM
  - 각 수면단계는 생리학적 신호 특성을 가짐

- 자동진단 AI 개발의 도전성
  - 블랙박스 문제
    - 결과에 책임 져야 함
    - AI는 자신의 판단에 대해 설명할 수 있어야 함
  - 멀티모달 신호의 통합 분석
    - 복잡한 멀티모달 시계열 신호 분석 AI 개발이 필요

### 3-2. 도메인 전문지식 주입 과정
- 핵심 아이디어1: 이미지 데이터 포멧 변환
  - 데이터의 종류는 시계열이지만, 의료인 관점에서 바라볼 때 이 작업의 본질은 컴퓨터 비전(이미지 판독)

- 핵심 아이디어2: 비전 트랜스포머 기반 신경망 학습
  - 의료 AI에서는 아직 CNN이 널리 쓰이나, 본 작업에는 어울리지 않음

### 3-3. 성능 평가
- 의료진이 이해할 수 있는 설명력
  - 어느 생체신호의 어느 시점을 보고 수면단계를 진단했는지 파악하고 시각화

- 에폭 간 맥락 파악 및 결과 보정

## 4. 개발 사례 2: 비접촉식 수면 무호흡증 진단 AI
### 4-1. 비접촉식 수면 무호흡증 진단 AI의 필요성
- 수면 적외선 영상
  - 수면 중 무의식적 미세 움직임을 분석해 수면 신호를 추출

### 4-2. 도메인 전문지식 주입 과정
- 수면 영상 데이터셋
  - 병원에서 수면다원검사를 진행하면서 영상을 동시 촬영
  - 접촉식 센서들로 추출한 생체신호와 수면영상이 시간 동기화: 상호 관련성 탐구에 적합

- 핵심 아이디어: 무호흡증 감지에서 호흡 각성 탐지로
  - 무호흡/저호흡의 움직임은 너무 미세해서 감지 어려움
  - 호흡 각성: 인체는 무호흡 상태 오래 지속되는 것 막기 위해 무의식적으로 몸을 크게 뒤척여 기도를 다시 개방함
  - 무호흡/저호흡 이벤트보다 짧은 시간 지속됨
  - 움직임이 큰 호흡 각성 먼저 감지하고, 그 전에 일어난 무호흡/저호흡 이벤트를 역추적하자

### 4-3. 성능 평가
