# 자연어처리 및 텍스트 파운데이션 모델 "거대 언어 모델(LLM)"
## 0. 파운데이션 모델
- 파운데이션 모델 이전: 새로운 테스크를 해결하려면 해당 테스크에 대한 "별도의 학습" 필요
- 파운데이션 모델: 새로운 테스크 해결하려면 "자세한 설명(프롬프트)"을 입력해주는 것으로 충분
- 파운데이션 모델의 3가지 구성요소
    1. 빅데이터
    2. 자가 학습(Self-supervised Learning)
    3. 어텐션(Attention) 기반 트랜스포머(Transformer) 모델

## 1. 텍스트 파운데이션 모델 살펴보기
### 1-1. 텍스트 파운데이션 모델 (거대 언어 모델)이란?
- 텍스트 파운데이션 모델의 특이점
    1. 규모의 법칙(Scaling Law): 더 많은 데이터, 큰 모델, 긴 학습 -> 더 좋은 성능
    2. 창발성(Emergent Property): 특정 규모를 넘어서면 갑자기 모델에서 발현되는 성질, ex) In-context learning, 추론(reasoning) 능력

### 1-2. 거대 언어 모델 예시
- 폐쇄형(Closed) 거대 언어 모델
    - ChatGPT, Claude, Gemini
    - 장점: 우수한 성능 및 최신 기능, 사용 쉬움
    - 단점: API 사용 시 마다 비용 발생, 모델이나 출력에 대한 정보가 제한적으로 제공
- 개방형(Open sourced) 거대 언어 모델
    - LLaMA, Gemma, Qwen
    - 장점: 무료로 다운로드 및 사용 가능, 모든 정보 공개
    - 단점: 충분한 계산 자원 필요, 성능 낮음

## 2. 거대 언어 모델의 학습
- 정렬(Alignment) 학습: 거대 언어 모델의 출력이 사용자의 의도와 가치를 반영하도록 하는 것
    - 지시 학습, 선호 학습
### 2-1. 지시 학습(instruction tuning)
- 주어진 지시에 대해 어떤 응답이 생성되어야 하는지
- 기존 언어 모델은 각 테스크마다 별도의 지도 추가 학습 및 모델을 저장 -> 모든 자연어 테스크는 텍스트 기반 지시와 응답으로 표현할 수 있지 않을까?
- 거대 언어 모델을 다양한 지시 기반 입력과 이에 대한 응답으로 지도 추가 학습
- 한계: 주어진 입력에 대해 적절한 하나의 응답이 있다고 가정 -> 정답이 정해져 있지 않은 개방형 테스크에서는 한계가 있음

### 2-2. 선호 학습(preference learning)
- 상대적으로 어떤 응답이 더 선호되어야 하는지
- 다양한 응답 중 사람이 더 선호하는 응답 생성하도록 추가학습
    1. 지시 학습을 통한 텍스트 파운데이션 모델의 추가 학습
    2. 사람의 선호 데이터 수집하여 보상 모델(Reward model, RM)을 학습
    3. 보상이 높은 응답을 생성하도록 강화 학습을 통해 추가 학습

## 3. 거대 언어 모델의 추론
### 3-1. 디코딩
- 거대 언어 모델의 자동회귀 생성(Auto-regressive Generation)
- 확률분포로부터 다음 토큰을 생성하는 알고리즘
    1. Greedy Decoding: 가장 확률이 높은 다음 토큰 선택
        - 장점: 사용하기 쉽다
        - 단점: 직후만 고려하기 때문에 생성 응답이 최종적으로 최선이 아닐 수 있다
    2. Beam Search: 확률이 높은 k개(beam size)의 후보를 동시에 고려
        - 장점: 좋은 응답 생성 확률 높다
        - 단점: 계산 비용 많이 늘어난다
    3. Sampling: 거대 언어 모델이 제공한 확률을 기준으로 랜덤하게 생성
        - 장점: 다양한 응답 생성
        - 단점: 생성된 응답의 품질 감소할 수 있음
    4. Sampling "with Temperature": 하이퍼파라미터 T를 통해 거대 언어 모델이 생성한 확률 분포를 임의로 조작
        - 장점: 창의성/안정성 조절 가능
        - 단점: T 높으면 품질 저하 낮으면 다양성 부족
    5. Top-K Sampling: 확률이 높은 K개의 토큰들 중에서만 랜덤하게 확률에 따라 샘플링
        - 장점: 품질이 낮은 응답을 생성할 가능성 줄일 수 있음
        - 단점: 확률 분포의 모양에 상관 없이 고정된 K개의 후보군 고려
    6. Top-P Sampling (or Nucleus Sampling): K 고정하는 대신, 누적 확률(P)에 집중하여 K 자동 조절
        - 장점: 품질, 다양성 균형
        - 단점: P값 설정 필요

### 3-2. 프롬프트 엔지니어링
- 입력 프롬프트 = 지시(instruction) + 예시(few-shot examples)
- Chan-of-Thought(CoT) 프롬프팅: 단순히 질문과 응답만 예시로 활용하는 것이 아니라 추론(reasoning) 과정도 예시에 포함

## 4. 거대 언어 모델의 평가와 응용
### 4-1. 거대 언어 모델의 평가
- 평가의 3가지 요소
    1. 목표: 시스템으로 무엇을 달성하고자 하는지
    2. 평가 방법: 어떤 방법으로 평가할 것인지
    3. 평가 지표: 어떻게 성공 여부를 판단할 것인지

- 거대 언어 모델 평가 방법의 종류
    1. 정답이 정해진 경우
        - 평가 방법: 예측과 정답을 비교하여 일치도 측정(정확도, Accuracy)
    2. 정답이 정해져 있지 않은 경우
        - 평가 방법1. 사람이 임의의 정답을 작성 및 이와 예측을 비교
        - 평가 방법2. 정답과 무관하게 생성 텍스트 자체의 품질만을 측정
        - 평가 방법3. 생성된 텍스트의 "상대적 선호"를 평가

### 4-2. 거대 언어 모델의 응용/한계
- 거대 언어 모델의 응용
    1. 멀티모달 파운데이션 모델
        - 핵심 아이디어: 다른 모달리티 데이터를 거대 언어 모델이 이해할 수 있도록 토큰화 및 추가 학습
    2. 합성 데이터 생성
        - Self-instruct: 사람이 만든 소량의 데이터를 기반으로 대규모 합성 데이터 생성

- 거대 언어 모델의 한계
    1. 환각(Hallucination)
        - 사실과 다르거나, 전적으로 지어낸 내용임에도 불구하고 정확한 정보와 동일한 자신감과 유창함으로 응답 생성
    2. 탈옥(Jailbreaking)
        - 프롬프트 엔지니어링을 통해 거대 언어 모델의 정렬을 우회할 수 있음
    3. AI 텍스트 검출
        - 거대 언어 모델의 무분별한 사용이 학교 및 회사에서 여러가지 새로운 문제 만들고있음
