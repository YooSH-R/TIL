## 딕셔너리 메서드(k=key, v=value)
- **D.get(k[, default])**
  - 키 연결된 값을 반환하거나 키가 없으면 None 혹은 기본 값을 반환
- **D.keys()**
  - 딕셔너리 키를 모은 객체를 반환(지연평가)
- **D.values()**
- **D.items()**
  - 딕셔너리 키/값 쌍을 모은 객체를 반환
  - (key, value)
- **D.pop(k[, default])**
  - 키를 제거하고 연결됐던 값을 반환(없으면 에러나 default를 반환)
- **D.clear()**
- D.setdefault(k[, default])
  - 키가 연결된 값을 반환
  - 키가 없다면 default와 연결한 키를 딕셔너리에 추가하고 default를 반환
- D.update([other])
  - other가 제공하는 키/값 쌍으로 딕셔너리를 갱신하고 기존 키는 덮어씀
```py
person = {'name': 'Alice', 'age': 25}
other_person = {'name': 'Jane', 'country': 'KOREA'}

person.update(other_person)
print(person)  # {'name': 'Jane', 'age': 25, 'country': 'KOREA'}

person.update(age=100, address='SEOUL')
print(person)  # {'name': 'Jane', 'age': 100, 'country': 'KOREA', 'address': 'SEOUL'}
```

## 딕셔너리의 확장
- defaultdict(자료형)
  - 딕셔너리에 존재하지 않는 키를 조회할 때, 자동으로 자료형의 기본값을 생성해주는 자료구조
  - 자주 쓰는 패턴
    1. 숫자 세기
    2. 그룹핑/리스트 모으기

## 세트 메서드(x=항목)
- **s.add(x)**
- s.update(iterable)
  - iterable이 그대로 들어가진 않고 요소들만 들어감
- s.clear()
- **s.remove(x)**
  - 항목 x 없을 경우 keyError
- s.pop()
  - 세트에서 **임의의** 요소를 제거하고 **반환**
- s.discard(x)
  - 세트에서 항목 x를 제거
  - remove와 달리 에러 없음
### 세트의 집합 메서드
- s1.difference(s2) = s1 - s2
- s1.intersection(s2) = s1 & s2
- s1.issubset(s2) = s1 <= s2
- s1.issuperset(s2) = s1 >= s2
- s1.union(s2) = s1 | s2 (합집합)

## 파이썬 문법 규격
- BNF
  - 프로그래밍 언어의 문법을 표현하기 위한 표기법
- EBNF
  - BNF를 확장한 표기법
  - 대표적 EBNF 메타기호
    - [] : 선택적 요소
    - {} : 0번 이상 반복
    - () : 그룹화
  
## 해시 테이블(hash table)
- 키(key)와 값(value)을 짝지어 저장하는 자료구조
- 해시 테이블의 원리
  1. 키를 해시 함수를 통해 해시 값으로 변환
  2. 변환된 해시 값을 인덱스로 삼아 데이터 저장, 찾음
  3. 이로 인해 검색, 삽입, 삭제를 매우 빠르게 수행

- 해시
  - 임의의 크기를 가진 데이터를 고정된 크기의 고유한 값으로 변환하는 것
  - 생성된 해시 값(고유한 정수)은 해당 데이터를 식별하는 '지문' 역할을 함
  - 파이썬에서는 이 해시 값을 이용해 해시 테이블에 데이터를 저장
  - 이 변환을 수행하는 것이 **해시 함수**
  - 정수 값은 해시 값이 숫자 자기 자신과 동일하거나 단순 계산으로 고정됨 -> pop() 순서 같음
  - 문자열은 해시 계산 시 파이썬의 해시 난수화(Hash Randomization)가 적용되므로, pop() 실행마다 순서가 달라질 수 있음
    - 해시 난수화: 문자열 해시 시, 파이썬 인터프리터 시작 때 설정되는 난수 시드(seed)가 달라질 수 있음
    - set의 pop()은 임의의 요소를 제거하고 반환
      - 버킷의 순서가 존재하기 때문에 "임의"라는 의미에서 무작위

- 해시 함수
  - 임의 길이 데이터를 입력 받아 고정 길이(정수, 해시 값)로 변환해 주는 함수.

- hashable
  - hash() 함수에 넣어 해시 값을 구할 수 있는 객체
  - 대부분의 불변 타입은 해시 가능
  - 가변형 객체는 기본적으로 해시 불가능
    - 값이 변하면 해시 값도 달라질 수 있어 해시 테이블 무결성(정확도)이 깨짐
  - 해시 테이블에는 hashable만 저장 가능
  - hashable 객체가 필요한 이유
    1. 해시 테이블 기반 자료 구조 사용
    2. 불변성을 통한 일관된 해시 값
    3. 안정성과 예측 가능성 유지
