## Git 동작
- git remote add origin remote_repo_url (origin: 원격 저장소의 별칭, url을 저장하는 변수)
- git push origin master: 원격 저장소에 commit 목록을 업로드(git아 origin이라는 원격 저장소에 master라는 이름의 브랜치를 push 해)
- git pull origin master: 원격 저장소의 변경사항만을 받아옴(업데이트)
- git clone remote_repo_url: 원격 저장소 전체를 복제(다운로드) **clone 받은 프로젝트는 이미 git init이 되어있음**
- 기타 명령어
  - git remote -v: 현재 로컬 저장소에 등록된 원격 저장소 목록 보기
- .gitignore: git에서 특정 파일이나 디렉토리를 추적하지 않도록 설정하는 데 사용되는 텍스트 파일
  - 이미 git의 관리를 받은 이력이 있는 파일이나 디렉토리는 나중에 gitignore에 작성해도 적용되지 않음(git rm --chached 명령어를 통해 git에서 캐시 삭제 필요)
- Revert & Reset
  - git revert: 특정 commit을 없던 일로 만드는 작업(push 전)
  - git reset: 특정 commit으로 되돌아가기(push 후)
  - git reset [옵션] <commit id>
  - remote에 올라간 commit을 reset하면 충돌할 수도 있으니 조심
