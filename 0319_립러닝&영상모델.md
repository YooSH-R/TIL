# CNN(Convolutional Neural Network) 살펴보기
## CNN vs. FCN(Fully-Connected Layer)
### FCN 모델
- 입력을 받아서 출력으로 변환하는 신경망의 기본 모듈

### CNN 모델
-  합성곱 레이어(Convolution Layer)
  - 입력 이미지를 필터와 연산하여 특징 맵(feature map)을 뽑아내는 모듈
  - Convolution: 필터를 이미지 상에서 이동시키면서 내적을 반복 수행, 내적으로 구한 모든 값을 출력 제공
  - 이미지를 1차원 구조로 변환하는 FCN과 달리 3차원 구조를 그대로 보존하면서 연산 -> 원래 이미지 형태 유지할 수 있게 함

## 모델구조
### CNN 모델 구조 - 수용영역(Receptive Feild)
- 중첩과 수용영역
  - CNN 레이어는 이미지 작은 부분인 지역정보를 추출하는데 유리하게 설계(필터 크기가 작기 때문)
  - 이미지 활용 작업에서 이미지 전체 의미 파악 필요하기 때문에 지역 정보 + 이미지 전체 맥락을 이해/활용하기 위한 설계 필요 -> Deep stacking, Pyramid structure, Pooling, Stride, Dilated Conv 등
  - 수용영역: CNN이 이미지 처리하면서 한 번에 볼 수 있는 영역 크기

## CNN 모델 구조 - 풀링
- 풀링: 효율적 연산 및 위치 변화의 강건성 확보
  - CNN 레이어의 출력을 줄여 연산 효율성을 확보: 입력 크기가 줄면 CNN의 연산량이 크게 줄어듦
  - 위치 변화 강건성: 입력 내 객체 위치가 다소 변해도 동일한 출력을 제공
  - 맥스풀링: 정해진 커널 사이즈로 이미지를 나누어 각 영역 내 가장 큰 값을 선택하는 연산
  
## CNN 모델 구조 - 스트라이드 합성곱
- 스트라이드 합성곱: 풀링 한계 개선
  - 필터를 스트라이드 값만큼 이동한 후 출력 연산
  - 풀링처럼 입력 해상도를 줄임(연산 효율과 위치 강건성)

# CNN 기반 모델 변천사
- AlexNet: GPU사용 + ReLU 등 -> CNN 대중화
- VGGNet: 단순하지만 깊은 CNN 구조 기준 제시
- ResNet: 깊은 네트워크에서 학습 안되는 현상 해결: 작은 모델의 성능 보장(Residual Block)
- MobileNet: 모바일/임베디드 환경에서 구동 가능하게 연산량, 모델상수 줄임(깊이별, 화소별 합성곱)

# 다양한 신경망 모델
## CNN의 한계
### CNN의 핵심 한계
- 데이터 순서(order)를 무시 -> 순차적(sequential) 데이터의 순서 정보를 반영하기 어려움
- 긴 거리 의존성 고려 부족: 이미지나 문장에서 멀리 떨어진 요소 간의 관계를 잘 학습하지 못함
- 픽셀 단위 복원 한계: 이미지 전체 정보 압축에 효과적이지만 위치 정보는 손실

## 시퀀스 데이터: RNN(Recurrent Neural Nets)
### RNN
- 순차적 데이터를 처리하기 위해 고안된 신경망 구조
  - 이전 단계 데이터를 은닉 상태(hidden state)로 다음 단계에 전달
  - 시계열/언어 데이터 적합

### 단순 RNN의 한계
- 기울기 폭발 혹은 소실 발생
  - 왜냐하면 역전파의 시퀀스 데이터가 적용되면서 선형가중치 W가 기울기 계산 시, 계속 곱셈으로 반영
- 장기기억 없음

### 대안 모델
- LSTM(Long Short-Term Memory)
  - 셀 상태(중요 정보 유지/기억), 게이트 유닛(정보의 흐름 제어)으로 구성
  - 구동철학: 긴 시퀀스 데이터에서 필요한 정보는 유지, 불필요한 정보는 지워 안정적 학습 유도
  - 기울기 전파 시 선형가중치의 곱 이외에 덧셈 형태로 전달하여 기울기를 장기적으로 보존 가능 but 정보 희석 문제의 한계


## 긴거리 의존성: 어텐션/ViT
### 어텐션 메커니즘
- 픽셀 거리와 상관없이 유사한 패치(이미지를 일정 크기로 잘라낸 작은 조각 단위)가 이미지에 존재 -> 관련 높은 패치/관련 낮은 패치 정보를 최종 결정에 반영: 쿼리(Q, 집중하려는 대상), 키(K, 다른 패치), 값(V, 키 패치의 정보), 어텐션(a, 쿼리와 키의 유사도), 출력(o, 어텐션으로 가감된 값의 총합)

### 셀프 어텐션(Self-Attention)
- 하나의 입력 안에서 패치 정보를 정의

### 교차 어텐션(Cross-Attention)
- 둘 이상의 이종 데이터에서 패치 정보를 정의

### 모델 별 특성 정리
- CNN: 지역적 특성에 집중, 장거리 맥락 이해 떨어짐, 데이터 순서 반영 불가
- RNN: 데이터 순서 반영, 장거리 맥락 학습 어려움
- LSTM: 데이터 순서 반영, 장거리 맥락 이해 가능, 정보 희석 문제
- Attention: 장거리 맥락에 탁월, 정보 희석 없음, 데이터 순서?

### ViT 위치 인코딩
- 인코딩 방식은 다양
- 패치 순서(위치) 정보 제공
  - 각 입력 토큰에 위치 정보를 담은 벡터를 추가

### ViT 학습 시 유의점
- 정규화 기법/데이터 증강 기술이 중요(막대한 상수 보유)
- 증류 학습(Knowledge Distillation) 기술이 효과적

### Vision Transformer의 트렌드
- CNN처럼 이미지의 계층적 이해를 활용할 수 없을까?
  - Swin Transformer: 이미지 전역에 직접 어텐션 수행하는 것은 너무 고비용, 비효율
    - 윈도우 영역을 지정, 그 내부의 어텐션에 지정
    - 윈도우를 shift해서 한번 더 설정
    - 1, 2가 중첩되면서 최종적으로 이미지 전역을 고려하는 효과

- Vision 백본, 데이터 확장성, 멀티모달 모델

# 이미지 모델 학습 전략
## 학습 전략의 중요성
### 좋은 모델 구조 = 우수한 성능?
- 좋은 구조만으로는 성능 보장 불가
- 학습에서 자주 겪는 문제들
  - 학습 불안정: 손실 폭발, 학습 멈춤, 수렴 실패
  - 과적합
  - 느린 수렴

### 학습 전략
- 구조 설계만큼 중요한 훈련 전략 제시
  - 학습률 스케줄링, 정규화, 초기화, 데이터 전처리

## 모델 구성
### 활성화 함수
- 입력 신호의 총합을 출력 신호로 변환하는 함수
  - 역할: 신경망에 비선형성 부여 -> 복잡한 패턴 학습 가능
  - Sigmoid: 0-1 사이의 값으로 출력 값 제어 -> 확률 값처럼 해석 가능
    - 입력이 매우 크거나 매우 작으면 출력이 거의 0이나 1에 고정
    - 출력 범위는 항상 양수 -> 편향된 업데이트 발생
  - Tanh
    - 데이터가 중심(0)을 기준으로 대칭 -> 학습 안정성 올라감
    - 기울기 소실 문제 여전 -> 학습 정체
  - ReLU
    - 양의 영역에서 포화되지 않음 -> 기울기 소실 문제 크게 줄어듦
    - 계산 효율성 높음
    - 학습 속도 빠름
    - 입력이 계속 음수면 뉴런이 0만 출력
  - Leaky ReLU
    - 음수 입력에도 작은 기울기를 부여
    - 음수 영역 기울기 값 설정의 임의성
    - 0 중심 대칭이 아님
  - Exponential Linear Unit(ELU)
    - 음수 영역에서의 포화 구간 제공 -> 기울기 소실 막고 왜곡문제 해결

### 데이터 전처리
- 이미지 조건을 일치
  - 학습/검증/테스트에 모두 동일하게 적용
    - 이미지 해상도, 색상, 밝기, 정규화
  - 일반적 정규화 방식
    - 평균0, 표준편차1로 정규화

### 모델 상수 초기화
- 0으로 초기화
  - 모든 출력은 0 -> 학습이 이루어지지 않는다
- 랜덤 초기화
  - 0을 중심으로 작은 랜덤 숫자
  - 깊은 모델에 적용 힘듦
- 자비에(Xavier) 초기화
  - 가중치 초기 분포의 분산을 입력 차원으로 맞춤
  - ReLU와 같이 쓸 때 문제
- 허(He) 초기화
  - 가중치 초기 분포의 분산을 2*입력 차원으로 맞춤
  - 비대칭, 유사 분산이 레이어 후반에도 유지

### 모델 정규화
- 학습과정에서 오차는 줄지만 검증 오차가 오른다면 -> 과적합의 신호 -> 정규화를 통해 검증 오차 줄이기
- 학습 과정에서 특정 가중치가 지나치게 커지는 현상 방지

- 가중치 감소
  - L2 정규화(Ridge, Weight Decay)
    - 큰 가중치에 제곱으로 패널티
    - 모든 가중치 골고루 작아짐
  - L1 정규화(Lasso)
    - 모든 가중치에 절댓값 크기만큼 동일하게 패널티
    - 중요하지 않은 가중치는 완전히 0이됨
  - Elsastic net(L1 + L2)

- 드롭아웃
  - 학습 과정에서 일부 뉴런을 확률적으로 끔(0으로 설정)
  - 장점: 매 학습 스텝마다 다른 네트워크 구조가 샘플링 되는 효과
  - 단점: 테스트 시에는 뉴런을 전부 사용
  - 해결책: 학습 시 살아남은 뉴런의 출력을 1/(1 - p)배 스케일링

## 학습 과정
### 학습 안정성 전략
- 학습 비율은 가장 중요한 하이퍼파라미터
  - 기본 전략: 학습 비율을 큰 값에서 epoch 지날 수록 작은 값으로 조정
- 과적합 방지를 위한 또 다른 방법: 빠른 종료(Early Stopping) 기법
  - 모델이 학습하는 동안 검증 데이터 셋을 통해 검증 손실 모니터링
  - 학습 진행하면서 성능 오히려 나빠지기 시작하면 학습 조기 중단

### 하이퍼파라미터 선정
- 파라미터: 학습을 통해 모델 스스로 얻는 값
- 하이퍼파라미터: 학습 전 사용자가 정해야 하는 값, 학습 진행되는 동안 고정
- 하이퍼파라미터 값 탐색 방식
  - 그리드 탐색
    - 후보들을 전부 조합
  - 랜덤 탐색
    - 무작위로 값 선택해 탐색
    - 성능에 크게 영향 주지 않는 축의 경우의 수를 실험하지 않아도 됨

- 체크리스트
  - 데이터 입력 등 문제 없는지: 초기 손실값 확인
  - 학습률과 초기화 확인: 작은 샘플을 활용, 과적합 시켜보기
  - 최적화 툴 활용, 학습률 찾기: 손실이 실제로 줄어드는 학습률부터 시작
  - 가중치 변형 방식과 좋은 조합 탐색: 1-5에폭 동안 모델 학습시켜 성능 비교
  - 10-20 에폭까지 추가 학습: 학습률 변경은 적용하지 않음
