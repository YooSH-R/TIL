# 7차시	
- Post-Training	Model
  - 거대 언어 모델 학습
  - pre-training: 방대한 텍스트 데이터를 이용한 학습을 통해 언어 패턴, 지식 등을 배운다 -> 다음 단어 예측
  - 유저의 의도를 파악하고 원하는 답변을 모델이 응답하도록 사후 학습 진행 -> 사람이 원하는 방식으로 대화, 안전하고 유용하게 만듦
  - ex) instruction-tuning, RLHF 등

- Instruction-tuning 특징	
  - 사전 학습 후 유저 의도와 일치 하지 않는 llm 해결 위해 파인 튜닝 진행
  - intsruction-tuning: 언어 모델이 사람이 내린 지시문(instruction)을 따르도록 학습하는 단계
  - 정답 레이블이 요구됨, 다양한 태스크를 풀 수 있도록 적응하는 것에 중점을 둠

- RLHF(Reinforcement Learning from Human Feedback) 파이프라인	
  - RLHF: instruction-tuning 한계 극복(정답 데이터 수집 곤란, 인간 선호와 불일치 등)
  - 언어모델의 응답 중 기대보상(expected reward)를 최대화 하는 것을 목표
  - 인간 선호도 모델링: 직접 점수 매기지 않고, 응답 비교

- RAG(Retrieval-augmented Generation) 정의
  - 사용자의 질문에 답하기 위해, datastore에서 관련 정보를 검색(Retrieval) 해와서, 이를 언어모델이 생성(Generation) 단계에 함께 활용하는 방법

- Information Retrieval(IR, 정보검색)의 종류
  - IR: 사용자의 질의(Query)에 맞는(가장 관련성 높은) 정보를 대규모 데이터에서 찾아 제공하는 과정
  - 활용: RAG
  - Retriever: IR수행하는 모듈, RAG 첫 단계 역할 수행
    - Sparse Retriever(어휘적 유사도 기반): 전통적 IR, 쿼리와 문서 간 정확한 용어 일치에 기반
      - TF-IDF: 문서 내 특정 단어의 중요도를 나타내는 가중치 방식
      - 장점: 단순성, 효율성, 투명성
      - 단점: 제한된 의미 이해, ex) 악당, 나쁜놈 매칭 x
    - Dense Retriever(의미적 유사도 기반): 쿼리와 문서 표현하기 위해 dense vector(embeddings)을 활용해 의미적 유사도에 기반
      - 임베딩 모델(ex. BERT) 사용하여 단어/문장을 의미 공간에 매핑
      - Bi-encoder: 쿼리와 문서를 별도의 인코더로 각각 임베딩한 후 두 벡터의 유사도로 관련성 계산, 대조 학습을 통해 학습, 쿼리가 긍정적 문서와 가깝게, 부정적 문서와 멀어지도록 유도, 빠르고 대규모 db 검색에 적합하지만 낮은 정확도
      - Cross-encoder: 두 개의 텍스트를 하나의 시퀀스로 결합, self-attention을 통해 모든 쿼리와 문서 토큰이 완전히 상호작용, 높은 정확도, 계산 비용 크고 처리 속도 느리다
      - 장점: 의미적 이해, 풍부한 쿼리 표현
      - 단점: 높은 연산 비용, 제한된 투명성, 데이터 및 모델 의존성


- Tool Usage in LLMs
  - 도구 사용: 텍스트 생성(text-generation mode) 모드, 도구 실행 모드(tool-execution mode) 간 전환
  - 도구 사용 유도법
    - 추론 시 프롬프트(inference-time prompting)
    - 학습(Tool Learning): 모방 학습, 인간의 도구 사용 행동 데이터를 기록함으로써 언어모델이 인간의 행동을 모방하도록 학습

- Context Protocol
  - Model Context Protocol(MCP): 언어 모델이 외부 툴과 상호작용하기 위한 표준화된 방식으로 정의한 프로토콜, 툴 호출, 응답 전달, 컨택스트 공유를 하나의 공통 규격으로 처리
  - 외부 툴 활용 연구 급증하면서 회사/모델 마다 각기 다른 툴 호출 방식 및 스키마 개발 -> 호환성 부족, 재사용 어려움
  - 아키텍쳐: Host, Client, Server
  - 계층(Layer): 데이터 계층, 전송 계층
  - 장점: 표준화, 확장성, 호환성, 재사용성, 투명성
